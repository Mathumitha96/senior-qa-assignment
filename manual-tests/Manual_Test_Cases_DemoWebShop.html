TC-001: User Registration

Field Details
Test Case ID TC-001
Test Case Title User Registration
Description Verify that a new user can successfully register and create an account on the Demo Web Shop
Test Steps
  1. Navigate to https://demowebshop.tricentis.com
  2. Verify homepage loads successfully
  3. Click on 'Register' link located in the top menu bar
  4. Verify registration page loads with all required fields
  5. Fill in the registration form with test data (see Test Data section)
  6. Verify all fields are filled correctly
  7. Click 'Register' button
  8. Wait for page to process the registration
  9. Observe system response
Test Data
✅ POSITIVE SCENARIO:
First Name: John
Last Name: Doe
Email: john.doe.test@example.com (use unique email)
Password: Test123!
Confirm Password: Test123!
❌ NEGATIVE SCENARIOS:
Test Case A - Invalid Email: Email: invalid-email (no @ symbol)
Test Case B - Weak Password: Password: 123; Confirm Password: 123
Test Case C - Mismatched Passwords: Password: Test123!; Confirm Password: Test456!
Test Case D - Duplicate Email: Email: existing.user@example.com (already registered)
Test Case E - Missing Required Fields: Leave First Name or Last Name blank
Expected Result
✅ POSITIVE:
  1. Registration page loads successfully
  2. All form fields accept input correctly
  3. User account is successfully created
  4. Success message 'Your registration completed' is displayed
  5. User is automatically logged in
  6. User is redirected to homepage
❌ NEGATIVE:
Test Case A: Error message displayed for invalid email format
Test Case B: Error message displayed for weak password
Test Case C: Error message displayed for password mismatch
Test Case D: Error message displayed for duplicate email
Test Case E: Error message displayed for missing required fields

TC-002: Search for Products

Field Details
Test Case ID TC-002
Test Case Title Search for Products
Description Verify that users can search for products using the search functionality and find relevant results
Test Steps
  1. Navigate to https://demowebshop.tricentis.com
  2. Locate the search box in the header/navigation area
  3. Verify search box is visible and accessible
  4. Enter search term from Test Data section
  5. Click 'Search' button (or press Enter key)
  6. Wait for search results page to load
  7. Review the search results displayed
  8. Verify search term is displayed on the results page
  9. Click on one of the search result products to view details
Test Data
✅ POSITIVE SCENARIO:
Search Term: 'book' (or 'computer', 'smartphone', 'laptop')
❌ NEGATIVE SCENARIOS:
Test Case A - Empty Search: Leave search field blank and click Search
Test Case B - Special Characters: Search Term: '@#$%^&*()'
Test Case C - Very Long Search Term: Search Term: 'a' repeated 500 times
Test Case D - Non-existent Product: Search Term: 'xyzabc123nonexistent'
Test Case E - SQL Injection Attempt: Search Term: 'book' OR '1'='1'
Expected Result
✅ POSITIVE:
  1. Search box is visible and functional
  2. Search results page loads successfully
  3. Relevant products matching search term are displayed
  4. Products show name, image, price, and 'Add to cart' button
  5. User can click products to view details
❌ NEGATIVE:
Test Case A: Error message or no results message displayed
Test Case B: Special characters handled safely, no errors
Test Case C: System handles long input gracefully
Test Case D: 'No products found' message displayed
Test Case E: SQL injection blocked, safe error message shown

TC-003: Add Product to Cart

Field Details
Test Case ID TC-003
Test Case Title Add Product to Cart
Description Verify that users can successfully add products to the shopping cart from product category pages and product detail pages
Test Steps
  1. Navigate to https://demowebshop.tricentis.com
  2. Browse to a product category by clicking on category from main navigation menu
  3. Verify category page loads and displays products
  4. Select product based on Test Data scenario
  5. Option A - Add from category page: Click 'Add to cart' button directly OR Option B - Add from product details: Click product name/image to view details page then click 'Add to cart'
  6. Verify 'Add to cart' button is visible and enabled
  7. Click 'Add to cart' button
  8. Wait for confirmation notification
  9. Verify notification message
  10. Click shopping cart icon to view cart
Test Data
✅ POSITIVE SCENARIO:
Product Category: Books (or Computers, Electronics)
Product: Any available in-stock product
Quantity: 1
❌ NEGATIVE SCENARIOS:
Test Case A - Out of Stock Product: Select product marked as 'Out of stock'
Test Case B - Invalid Quantity: Enter quantity as 0 or negative number
Test Case C - Very Large Quantity: Enter quantity as 99999
Test Case D - Disabled Product: Select product with 'Add to cart' button disabled
Test Case E - Expired Product: Select product that is no longer available
Expected Result
✅ POSITIVE:
  1. Category page loads with products displayed
  2. Product details page displays all information correctly
  3. 'Add to cart' button is visible and clickable
  4. Success notification appears: 'The product has been added to your shopping cart'
  5. Shopping cart icon shows updated item count
  6. Cart page displays product with correct details and price
❌ NEGATIVE:
Test Case A: Error message: 'Product is out of stock' or button disabled
Test Case B: Error message for invalid quantity or quantity reset to 1
Test Case C: Error message for quantity limit exceeded or system caps at maximum
Test Case D: Button remains disabled, no action possible
Test Case E: Error message: 'Product no longer available' or redirect to product list

TC-004: Price Calculation Verification

Field Details
Test Case ID TC-004
Test Case Title Price Calculation Verification
Description Verify that price calculations are accurate when multiple products with different quantities are in the shopping cart, including subtotals and total cart value
Test Steps
  1. Add first product to cart with test data from Test Data section
  2. Navigate back and add a second different product to cart
  3. Navigate to shopping cart page
  4. Verify both products are displayed
  5. Note unit prices of both products
  6. Update quantities based on Test Data scenario
  7. Click 'Update shopping cart' button
  8. Wait for cart to refresh
  9. Verify calculations for each product
  10. Verify total cart value
Test Data
✅ POSITIVE SCENARIO:
Product 1: Unit Price $10.00, Quantity 2, Expected Subtotal $20.00
Product 2: Unit Price $15.00, Quantity 1, Expected Subtotal $15.00
Expected Cart Total: $35.00
❌ NEGATIVE SCENARIOS:
Test Case A - Zero Quantity: Set quantity to 0 for one product
Test Case B - Negative Quantity: Enter -5 as quantity
Test Case C - Decimal Quantity: Enter 2.5 as quantity
Test Case D - Very Large Quantity: Enter 10000 as quantity
Test Case E - Decimal Prices: Product with price $9.99, quantity 3, verify calculation $29.97
Expected Result
✅ POSITIVE:
  1. Cart displays all products in table format
  2. Quantity input field accepts and updates correctly
  3. First product subtotal: $10.00 × 2 = $20.00 (correct)
  4. Second product subtotal: $15.00 × 1 = $15.00 (correct)
  5. Cart Total: $35.00 (sum of all subtotals)
  6. All calculations mathematically accurate
  7. Currency format displayed correctly
❌ NEGATIVE:
Test Case A: Product removed from cart or error message displayed
Test Case B: Error message displayed or quantity reset to 1
Test Case C: Error message for invalid quantity or decimal rounded/truncated
Test Case D: Error message for quantity limit or system caps at maximum allowed
Test Case E: Decimal calculation accurate: $9.99 × 3 = $29.97 (no rounding errors)

TC-005: Complete Checkout and Place Order

Field Details
Test Case ID TC-005
Test Case Title Complete Checkout and Place Order
Description Verify that a logged-in user can complete the entire checkout process from cart to order confirmation, including billing address, shipping, payment method selection, and order placement
Test Steps
  1. Navigate to https://demowebshop.tricentis.com and ensure logged in
  2. Verify user is logged in
  3. Add at least one product to cart if empty
  4. Navigate to shopping cart page
  5. Verify cart contents are correct
  6. Click 'Checkout' button
  7. Fill in billing address using Test Data
  8. Verify 'Ship to the same address' checkbox
  9. Click 'Continue' to shipping method
  10. Select shipping method
  11. Click 'Continue' to payment method
  12. Select payment method
  13. Click 'Continue' through payment information
  14. Review order summary
  15. Click 'Confirm' or 'Place Order' button
  16. Verify order confirmation page
Test Data
✅ POSITIVE SCENARIO:
First Name: John
Last Name: Doe
Email: john.doe@example.com
Country: United States
State: California
City: Los Angeles
Address: 123 Main Street
Zip: 90001
Phone: 555-123-4567
Shipping Method: Ground
Payment Method: Cash On Delivery
❌ NEGATIVE SCENARIOS:
Test Case A - Invalid Email: Email: invalid-email-format
Test Case B - Missing Required Fields: Leave First Name or Zip Code blank
Test Case C - Invalid Zip Code: Zip: 'ABC123' (non-numeric)
Test Case D - Invalid Phone: Phone: '123' (too short)
Test Case E - Invalid Address: Address: Only special characters '@#$%'
Test Case F - Empty Cart: Attempt checkout with empty cart
Expected Result
✅ POSITIVE:
  1. Checkout page loads with all sections visible
  2. All billing address fields accept input correctly
  3. Shipping and payment methods can be selected
  4. Order summary displays correctly with all details
  5. Order is processed successfully
  6. Order confirmation page displayed with order number
  7. Order appears in order history
❌ NEGATIVE:
Test Case A: Error message for invalid email format
Test Case B: Error message for missing required fields, cannot proceed
Test Case C: Error message for invalid zip code format
Test Case D: Error message for invalid phone number
Test Case E: Error message for invalid address format
Test Case F: Error message: 'Your cart is empty' or redirect to cart page
